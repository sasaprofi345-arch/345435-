# Настройка проекта в Roblox Studio

Ниже — список объектов, которые нужно создать в иерархии игры,
чтобы скрипты `FreddyAI.server.lua` и `Flashlight.client.lua`
заработали без правок.

## 1. ReplicatedStorage

Создайте здесь два `RemoteEvent` (ровно с такими именами):

```
ReplicatedStorage
├── FlashlightHit     (RemoteEvent)   -- клиент -> сервер, сообщает о попадании луча
└── JumpscareEvent    (RemoteEvent)   -- сервер -> клиент, запускает эффекты скримера
```

## 2. Workspace

### 2.1. Модель врага

```
Workspace
└── Freddy            (Model)
    ├── HumanoidRootPart  (Part)     -- PrimaryPart модели (важно!)
    │   ├── LaughSound    (Sound)    -- смех Фредди
    │   └── DropSound     (Sound)    -- звук падающего предмета
    └── ... остальные части / меш Фредди
```

Требования:

- `Model.PrimaryPart` обязательно установлен в `HumanoidRootPart`,
  иначе `PivotTo` не сможет переместить модель целиком.
- Все части модели можно объединить через `WeldConstraint` к
  `HumanoidRootPart`, чтобы они двигались вместе.
- `HumanoidRootPart.Anchored = true`, чтобы Фредди не падал и не
  улетал от физики между телепортами.
- `CanCollide` у частей — на ваше усмотрение; для классической
  механики JoC проще оставить `false`, чтобы игрок не застревал.
- `Sound.SoundId` — заполните своими ассетами (`rbxassetid://...`),
  `Sound.RollOffMode = InverseTapered`, `EmitterSize` ~ 10 для
  эффекта "слышно издалека, но направление улавливается".

### 2.2. Точки телепортации

```
Workspace
└── FreddyTeleportPoints   (Folder)
    ├── Point1   (Part)    Anchored, CanCollide = false, Transparency = 1
    ├── Point2   (Part)    ...
    ├── Point3   (Part)
    └── ...                сколько угодно точек
```

- Имена точек значения не имеют — скрипт просто берёт `:GetChildren()`.
- Расставьте парты по интересным местам локации (углы коридоров,
  тёмные комнаты, под лестницами).
- Сделайте парты невидимыми (`Transparency = 1`), а `CanCollide = false`
  чтобы игрок не натыкался на них.

## 3. ServerScriptService

```
ServerScriptService
└── FreddyAI           (Script)   -- сюда положите содержимое FreddyAI.server.lua
```

## 4. StarterPlayer → StarterPlayerScripts

```
StarterPlayer
└── StarterPlayerScripts
    ├── Flashlight          (LocalScript) -- содержимое Flashlight.client.lua
    └── SurvivorMovement    (LocalScript) -- содержимое SurvivorMovement.client.lua
```

> `SurvivorMovement` — DBD-подобный мувмент для R6: спринт со
> стаминой (Shift), присед (C), быстрый разворот (Q), look-back
> на удержание (LeftAlt) и контекстный vault (Space у низких
> препятствий). Все параметры — в верхнем блоке `CONFIG`.

## 5. Управление

- Игрок включает / выключает фонарик клавишей **F**.
- Луч испускается из камеры, поэтому фонарь "светит туда, куда смотрит игрок".
- При прямом наведении луча на любую часть модели `Freddy` сервер
  получает событие, играет скример и телепортирует врага.

## 6. Тонкая настройка (по желанию)

В верхней части `Flashlight.client.lua` можно подкрутить:

| Параметр              | Значение по умолчанию | Что делает                                  |
| --------------------- | --------------------- | ------------------------------------------- |
| `FLASHLIGHT_RANGE`    | 80                    | максимальная дальность луча в studs         |
| `FLASHLIGHT_ANGLE`    | 25                    | половинный угол конуса SpotLight (град.)    |
| `DETECTION_INTERVAL`  | 0.1                   | как часто проверять raycast (сек.)          |

В `FreddyAI.server.lua`:

| Параметр                | Где задаётся                   | Что делает                          |
| ----------------------- | ------------------------------ | ----------------------------------- |
| Интервал между телепортами | `math.random(6, 12)`        | разброс пауз между перемещениями    |
| Шанс ambient-звука      | `if math.random() < 0.4`       | вероятность сыграть смех/падение    |
| Макс. дистанция проверки | `if distance > 80`            | анти-чит на дальность фонаря        |

## 7. Что происходит на практике

1. Игрок входит в тёмную локацию, у него фонарик (`F` — вкл/выкл).
2. Фредди стоит на одной из точек. Иногда смеётся / роняет предметы.
3. Игрок ориентируется на звук, поворачивает камеру в ту сторону.
4. Если луч попал в модель `Freddy`:
   - клиент шлёт `FlashlightHit` на сервер,
   - сервер валидирует расстояние,
   - вызывает `JumpscareEvent` у клиента (тряска + красный фильтр),
   - на 0.25 сек "телепортирует" Фредди вплотную к лицу игрока,
   - и тут же убегает в новую случайную точку.
5. Цикл продолжается.
